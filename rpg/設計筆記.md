  - 註:舊存檔的舊格式符文（無 `mul` 旗標）會被當加法型處理,不影響運作。

## C. 大方向 / 需要拿捏的旋鈕

- ⬜ **基礎值 vs 詞綴「對不上」**:兩者的樓層權重不同(基礎值每層 +0.9、詞綴每層 +0.3),導致「深樓層低輪迴」vs「淺樓層高輪迴」的裝備,基礎值與詞綴各說各話。→ 對齊樓層權重,讓兩者一體。
- ✅ ~~**裝備 vs 怪物輪迴倍率落差**:`CYC_VAL` → `[0.8, 2.8, 6.5]`~~ → **已被 v367 的 cycK 制取代(見下)**。
- ✅ **輪迴裝備縮放重做(v367,cycK 制)**:舊制 `×(1+CYC_VAL)` 整值乘,會把詞綴的「骰值常數」一起放大 → 詞綴 8 層就爆高、基礎值 34+ 層才跟上,基礎值與詞綴脫鉤(實測體感:畢業裝直推、掉落沒有換的理由)。改為 **`CYC_K=[1.5, 4.3, 12.3]`(≈0.79×CYC_MULT)只乘「樓層項」**:基礎值=`6+0.9·f·K`(等效樓層 `f×K` 進 CURVE)、素質詞綴=`骰值+0.3·f·K`、hp/mp 無樓層項維持整值×K。效果:每重輪迴換裝交叉點自動落在 **36–44 層**(I:44/II:39/III:36,未精煉橙 vs 前輪畢業+12),詞綴交叉 33 層與基礎值同步;裝備仍追不上怪物(I-100 裝×2.4 vs 怪×3.7),貪婪落差保留;無限段裝 ×2.3/重 vs 怪 ×2.86/重=牆。祝福縮放拆成 `BLESS_CYC`(凍結舊值)與裝備脫鉤;重鑄改傳 cyc 進 `rollAffixVal` 統一縮放,不再外掛二次乘。護甲交叉 ~53 層偏晚(精煉+21 對小基數佔比大),**觀察後再調**。舊裝備數值已骰定,無需遷移。
- ⬜ **基礎值 vs 詞綴樓層權重**(較小的殘留):base 每層 +0.9、詞綴每層 +0.3,跨樓層比較仍會不同步。這次只動了輪迴落差,樓層權重對齊留待之後(調高詞綴的 `floor×0.3` 係數)。

- ⬜ **職業特色 = 可玩空間目標**:四職對應四種「活法」,必要時調整技能與升級方向(`SKILLS` / `SKILL_UPS`)。
  - 根源問題:高輪迴下只有「率(閃避/防禦率)」會隨怪物傷害縮放,血量/護甲/格擋是死值被輾 → 逼所有人疊率。要讓其他防線也能隨傷害縮放,才有多樣性。
  - 取捨:難度越硬 → 能活的 build 越少;目標是「夠難、但不只一條路能過」。

| 職業 | 特色骨幹 | 高輪迴要調的 |
|------|----------|--------------|
| 盜賊 ✅ | 閃避 + 疊毒收割 | 毒改20(見A)+ **內建毒吸 25%**(`ASSASSIN_POISON_LEECH`,已做) |
| 劍士 ⬜ | 格擋硬吃 + 處決 | 讓**格擋隨體力/傷害縮放**(否則死值被輾)——待定公式 |
| 法師 ⬜ | 爆發速殺 + 無視格擋 | 確保爆發**追得上怪血** + 屏障擋一下——待定 |
| 制魔師 ⬜ | 吸取續戰 + 疊 debuff | 對**高血怪**微調,讓消耗流打得動——待定 |

### 平衡第一包 A/B(v368,與 C 的 cycK 制同版本組)
- ✅ **A:DOT 首領承傷閥**:玩家施加的毒/燃,每回合每種上限=敵生命上限×capRate——雜魚(含王隨從)無上限、精英(域限/一般)與小王 8%、域主/最終王 5%。唯一入口 `dotHitEnemy(e,kind,raw)`+`dotCapRate(e)`(battle.js),三個跳傷點(敵方毒跳/`tickBurn`敵側/催毒 poisonProc)全部導過去,**禁止散寫**。額度掛敵人物件 `e._dotUsed`,`startPlayerTurn` 重置(週期=玩家回合起手)。層數照疊(餵絞殺/腐燃、延長受頂回合),只封跳傷;敵方對玩家的毒燃不變。UI:敵方狀態列超閥層數灰字(`statusHtml` 第三參數)、tooltip 註明、跳字顯示截斷後實際值。
- ❌ **B:吸取回復節流閥——實作後撤回(v369),不採用**。原規格「每回合回復 ≤ 15%(盜賊20%)生命上限」以**生命上限**為基準,而生命上限正是高輪迴唯一不吃 cycMult 的數值 → 實測基準(輪迴I 100層:血量 300、每回合承傷 ≈314、雜魚戰需 5 回合)下,額度僅 45/回合,等於直接刪除盜賊。**結論:吸血已有 `VAMP_CAP` 一道彙總上限,第二道閥與它對掐(前者管回血相對「己方輸出」、後者管相對「敵方輸出」),必有一個是裝飾品。決定只留 VAMP_CAP。**
  - 📌 **病灶另案**:高輪迴的過剩續航,根因是「血量/護甲/格擋不吃 cycMult,只有率與按比例回血能跟上」→ 吸血成為唯一活路,所以砍不得。真正該做的是**讓血量或減傷有一條吃輪迴的成長軸**(見 C 段既有條目),不是在 leech 端加閥。第二包主題改為此項,D 案(防禦力平減 ×0.5)僅為權宜、優先度下修。
- 📌 已知取捨(A 帶來的):催毒/劇毒/烈焰/蝕魂毒傷加成對「有閥敵人」邊際歸零(raw 遠超 cap),只對雜魚有感——待實測後決定是否給催毒獨立小額度。

## F. 「成功回營才留存」集中化(核心原則)
